## 1. 청사진 사용법

- 이 문서는 각 셀마다 "어떤 문항을 만들면 가장 안정적으로 그 셀을 측정할 수 있는지"를 제안하는 기본 설계도다.
- 실제 문항 메타데이터에는 이 문서의 `대표 문항 유형`과 `권장 제시문`을 우선 반영하되, 필요하면 같은 셀 안에서 변형할 수 있다.
- 단, `정답 판단 기준`이 흔들리면 다른 영역이나 다른 단계 문항이 되므로 반드시 유지한다.

## 3. 영역별 제작 메모

### 3.1 IDE

- 단계 상승의 핵심은 `입장 밝히기 → 이유 → 목적 적합성 → 타당성 → 논리 판단 → 반론 고려 → 다관점 종합`이다.
- 정답은 늘 "가장 그럴듯한 내용"이 아니라 "해당 글쓰기 과업에 가장 기능적으로 맞는 내용"이어야 한다.

### 3.2 ORG

- 단계 상승의 핵심은 `문장 연결 → 문단 완성 → 3단 구조 → 전개 방식 → 설명/보고 구조 → 형식화 → 해석 기반 구조화`이다.
- 조직 문항은 내용 지식보다 흐름과 구조 판단을 묻는 데 집중해야 한다.

### 3.3 EXP

- 단계 상승의 핵심은 `감정 표현 → 독자 고려 → 목적·독자 맞춤 표현 → 효과적 표현 → 사회적 맥락 이해 → 맥락 조정 → 복합 매체 설계`이다.
- 같은 내용을 두고 표현만 비교하는 방식이 가장 안전하다.

### 3.4 CVN

- 단계 상승의 핵심은 `표기 정확성 → 문장 부호 → 문법 → 문장 구조 → 문체·격식 → 간결성 → 일관성`이다.
- 상위 단계로 갈수록 단일 규칙보다 전체 문장과 글의 품질을 보게 해야 한다.

### 3.5 REV

- 단계 상승의 핵심은 `단일 오류 수정 → 주제 이탈 수정 → 복합 오류 수정 → 다기준 수정 → 과정 인식 → 전략 조정 → 종합 수정`이다.
- 상위 단계에서 `REV`는 "무엇이 틀렸나"보다 "어떻게 고쳐야 글의 질이 가장 좋아지는가"를 묻는다.

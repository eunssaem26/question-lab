# 브런치 × 네이버 블로그 크로스포스팅 가이드

## 발행 순서 (추천)

1. **자체 블로그** (Astro/Vercel) — 원본, canonical URL
2. **브런치** — 발행 후 1~2일 뒤
3. **네이버 블로그** — 브런치와 같은 날 또는 하루 뒤

> 검색엔진이 원본 URL을 먼저 인덱싱하도록 시차를 두는 것이 좋습니다.

---

## 브런치 발행 체크리스트

- [ ] 브런치 작가 신청/승인 확인 (brunch.co.kr)
- [ ] "질문연구소는 이렇게 시작되었습니다" 브런치북(시리즈) 생성
- [ ] 각 편 본문 복사 → 브런치 에디터에 붙여넣기
- [ ] 커버 이미지 업로드 (각 글 상단에 표기된 이미지)
- [ ] 본문 중 [이미지] 위치에 해당 캐릭터 이미지 삽입
- [ ] 해시태그 입력 (각 글 하단에 표기)
- [ ] 브런치북에 순서대로 추가
- [ ] "원문 보기" 링크로 Astro 블로그 URL 추가

## 네이버 블로그 발행 체크리스트

- [ ] 네이버 블로그 개설 (blog.naver.com)
- [ ] "질문연구소 설립기" 카테고리 생성
- [ ] 각 편 본문 복사 → 스마트에디터에 붙여넣기
- [ ] 제목/본문의 **볼드**, 인용구, 구분선 수동 적용
- [ ] 이미지 삽입 (본문 중 [이미지] 위치)
- [ ] 태그 입력 (각 글 하단에 표기)
- [ ] 시리즈 설정으로 5편 묶기
- [ ] "원문 보기" 링크 추가

## 플랫폼별 핵심 차이

| 항목 | 브런치 | 네이버 블로그 |
|------|--------|-------------|
| 제목 | 짧고 문학적 (제목+부제) | 키워드 포함 SEO형 |
| 표 | 미지원 → 텍스트로 변환 | 지원 → 표 그대로 사용 |
| 코드블록 | 미지원 → 인용구로 변환 | 인용구/강조상자로 변환 |
| 해시태그 | 3~5개 | 10~15개 |
| 톤 | 에세이, 여백 활용 | 정보 전달, 스캔하기 쉽게 |
| CTA | 부드러운 마무리 | 공감/댓글/이웃 유도 |

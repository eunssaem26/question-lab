## 1. 공통 출제 원칙

### 1.1 교육과정 정합성
- 모든 문항은 평가준거틀의 35셀 매트릭스(7단계 × 5영역)에 근거하여 출제한다.
- 각 문항은 반드시 하나의 성취기준 코드와 연결된다.
- 각 문항은 반드시 하나의 **주 측정 영역(primary_domain)** 을 가진다. 성취기준이 둘 이상의 영역과 연결되더라도 통계 집계와 단계 판정은 주 측정 영역 기준으로 처리한다.
- 매트릭스의 "평가 내용"이 문항이 측정하는 능력을 정확히 반영해야 한다.

### 1.2 단계 적합성
- 제시문(학생 글, 글쓰기 상황)의 어휘 수준, 문장 길이, 주제 복잡도가 해당 단계에 적합해야 한다.
- 상위 단계 문항은 전체적으로 하위 단계 문항보다 높은 인지 부담을 보여야 한다. 다만 파일럿 결과에서 일부 개별 문항의 난도가 교차할 수 있으므로, 최종 판단은 문항 단건이 아니라 단계 묶음 기준으로 한다.
- 같은 학년군 내 기초/심화 단계 간 질적 차이가 문항에 드러나야 한다.

### 1.2A 위계 판단의 기준 축
- 단계 차이는 단순히 "어려운 어휘를 썼는가"로 판단하지 않는다.
- 출제자는 아래의 기준 축 중 무엇이 상향되는 문항인지 분명히 설명할 수 있어야 한다.

| 영역 | 위계가 올라가는 핵심 기준 |
|------|--------------------------|
| IDE | 의견 제시 → 이유 제시 → 주장/근거 구분 → 복수 근거의 타당성 판단 → 논리 관계 판단 → 반론 고려 → 다관점 종합 |
| ORG | 문장 연결 → 문단 완성 → 3단 구조 → 전개 방식 선택 → 설명/보고 구조화 → 형식화된 보고 조직 → 복합 구조와 결과 해석 |
| EXP | 감정 표현 → 독자 고려 → 목적·독자에 맞는 표현 선택 → 비유/묘사·매체 활용 → 사회적 소통 맥락 이해 → 맥락 맞춤 조정 → 복합 매체 설계 |
| CVN | 표기 정확성 → 문장 부호·기본 문법 → 높임·호응 → 접속·피동·사동 → 문체·격식 적합성 → 간결성·명확성 → 글 전체의 일관성 |
| REV | 단일 오류 수정 → 주제 이탈 수정 → 복합 오류 수정 → 내용·조직·표현 기준 적용 → 쓰기 과정 인식 → 전략 조정 → 종합적 재설계 |

### 1.2B 성취기준 정합성 판단 원칙
- 성취기준 코드는 "문항 소재"가 아니라 "문항이 직접 측정하는 수행"에 맞춰 부여한다.
- 하나의 성취기준이 여러 영역과 닿더라도, 문항의 정답 판단 근거가 되는 능력은 하나여야 한다.
- 특히 다음 혼동을 피한다.
  - `[6국03-02]`는 내용 자체보다 "목적이나 주제에 따라 알맞은 내용과 매체를 선정"하는 기준이므로, 문항이 "무엇을 쓸까"를 묻는지 "어떻게 표현할까"를 묻는지 구분해 `IDE` 또는 `EXP`를 정한다.
  - `[9국03-04]`는 주장과 근거의 타당성을 묻는 성취기준이므로 기본적으로 `IDE` 중심이다. 문단 배열이나 논증 구조 배치는 보조 요소일 뿐이다.
  - `[9국03-05]`는 성찰적 글감과 의미 형성이 핵심이므로, 단순 문체 선택 문항으로 처리하지 않는다.
  - `[9국03-07]`은 단순한 맞춤법 교정보다 쓰기 과정 점검과 조정 전략을 묻는 문항에서 우선 사용한다.

### 1.3 CBT 측정 전략: 판단력을 통한 간접 측정
- 글쓰기는 본질적으로 **수행(production)** 역량이지만, v1 CBT에서는 **판단(judgment)** 역량을 통해 간접 측정한다.
- 이 접근법은 SAT Writing, ACT English 등 검증된 대규모 평가에서 널리 사용된다.
- **v2 확장 계획**: 서술형 문항(문장·문단 수준 글쓰기)과 AI 자동 채점을 도입하여 실제 글쓰기 수행 능력을 직접 측정할 예정이다.

| 측정 방법 | 설명 | 해당 영역 |
|----------|------|----------|
| 최선의 수정안 선택 | 주어진 글에서 가장 적절한 수정안을 고르기 | REV, CVN, EXP |
| 오류 발견 | 문장이나 문단에서 오류를 찾기 | CVN, REV |
| 내용 선정 | 목적에 가장 알맞은 내용 고르기 | IDE |
| 순서 배열 | 문장이나 문단의 논리적 순서 배열 | ORG |
| 표현 선택 | 독자/목적에 맞는 표현 고르기 | EXP |
| 연결어 선택 | 문맥에 맞는 연결어 고르기 | ORG |

### 1.4 공정성
- 특정 성별, 지역, 문화적 배경에 유리하거나 불리한 내용을 배제한다.
- 제시문 주제는 학생이 사전 지식 없이도 글만으로 답할 수 있어야 한다.
- 선택지에 학생을 혼란스럽게 하는 불필요한 함정을 넣지 않는다.

### 1.5 학년 라벨 미사용
- 문항, 선택지, 해설에 "초3", "중1" 등의 학년 표현을 사용하지 않는다.
- 단계 번호(1~7단계)로만 수준을 표현한다.

### 1.6 읽기 문항과의 독립성
- 글쓰기 문항은 읽기 문항과 독립적으로 출제한다.
- 글쓰기 문항에서 읽기 이해력이 정답을 좌우해서는 안 된다. 제시문은 해당 단계 학생이 충분히 이해할 수 있는 수준이어야 한다.
- 같은 지문을 읽기와 글쓰기 문항에서 공유하지 않는다.

## 2. 제시문 작성 기준

### 2.1 제시문 유형

글쓰기 문항의 제시문은 읽기 문항과 다르다. 읽기는 "완성된 글"을 읽히지만, 글쓰기는 "글쓰기 상황"을 제시한다.

| 유형 | 설명 | 주요 사용 영역 |
|------|------|--------------|
| **학생 글 초안** | 학생이 쓴 글(또는 글의 일부)을 제시 | REV, CVN, ORG |
| **글쓰기 상황** | 글을 쓸 목적, 독자, 상황을 제시 | IDE, EXP |
| **개요/메모** | 글쓰기 계획, 개요, 브레인스토밍 메모를 제시 | IDE, ORG |
| **수정 전 글** | 고쳐쓰기 전의 글을 제시 | REV, CVN, EXP |
| **대화/상황문** | 글을 쓰게 된 배경 대화나 상황을 제시 | EXP, IDE |

### 2.2 제시문 길이 기준

| 단계 | 글자 수 | 비고 |
|------|---------|------|
| 1 | 50~150자 | 짧은 문장 또는 한 문단 수준의 학생 글 |
| 2 | 100~300자 | 한두 문단 수준의 학생 글 |
| 3 | 200~400자 | 서론-본론-결론 구조가 보이기 시작하는 글 |
| 4 | 250~450자 | 전개 방식이 드러나는 글 |
| 5 | 300~500자 | 설명문, 보고서 등 장르 특성이 드러나는 글 |
| 6 | 350~550자 | 논증 구조가 있는 글, 보고서 형식의 글 |
| 7 | 400~600자 | 복잡한 구조, 다양한 설명 방법이 사용된 글 |

- 위 기준은 **CBT 운영용 기본값**이다.
- 1차 라우팅용 제시문은 읽기 부담이 글쓰기 판단을 가리지 않도록 가능하면 **450자 이하**로 제한한다.
- 2차 확인이나 파일럿 연구에서만 상한을 예외적으로 늘릴 수 있으며, 그 경우에도 학생이 제시문을 반복해서 읽지 않아도 풀 수 있도록 문항 구조를 단순화한다.

### 2.3 글쓰기 과제 유형 배분 (단계당)

| 과제 유형 | 편수 | 비고 |
|----------|------|------|
| 의견/주장 글 | 2~3편 | 전 단계 필수 포함 |
| 설명문 | 1~2편 | 3단계 이상에서 비중 증가 |
| 보고서 | 1편 | 5단계 이상에서 도입 |
| 감상/성찰문 | 1~2편 | 전 단계 포함 가능 |
| 편지/안내문 | 0~1편 | 독자 고려 평가에 활용 |

### 2.4 글쓰기 주제 영역

- **1~2단계**: 일상생활 중심 — 학교생활, 가족, 친구, 취미, 자연
- **3~4단계**: 교과 연계 확대 — 환경, 과학, 역사, 사회 문제 (단순)
- **5~7단계**: 사회적 쟁점 확대 — 환경, 기술, 미디어, 사회 문제, 자기 성찰
- 전 단계 공통: 제시문 내부 정보의 충분성과 공정성을 우선한다.

### 2.5 제시문 작성 시 유의점
- 모든 제시문은 AI 생성으로 작성한다.
- 실제 교과서나 저작물의 직접 인용은 하지 않는다.
- 학생 글 초안은 해당 단계 학생이 실제로 쓸 법한 수준으로 작성한다.
- 의도적으로 넣는 오류(어법 오류, 조직 오류 등)는 해당 단계에서 발견할 수 있는 수준이어야 한다.
- 한 제시문에서 최대 3개 영역의 문항을 출제할 수 있다.

### 2.6 제시문 식별자 규칙
- 형식: `WP-{단계}-{번호}`
- 예시: WP-1-001 (1단계 글쓰기 제시문 1번), WP-3-002 (3단계 글쓰기 제시문 2번)
- 읽기 지문(`P-`)과 구분하기 위해 `WP-` 접두사를 사용한다.

## 3. 문항 작성 기준

### 3.1 문항 유형별 규칙

#### 4지선다
- 선택지 4개. 정답 1개, 오답(매력적 오답) 3개.
- 선택지 길이가 비슷해야 한다 (정답만 길거나 짧으면 안 됨).
- 선택지 배열: 길이순 또는 논리적 순서. 정답 위치를 고르게 분산.
- "모두 맞다", "없다" 같은 포괄적 선택지 지양.

#### 순서 배열
- 글 조직 영역에서 주로 사용.
- 배열할 항목 3~4개 (문장 또는 문단).
- 발문 예시: "다음 문장을 글의 흐름에 맞게 순서대로 배열하시오."

#### 짝짓기형
- 어법과 규범 영역에서 사용 가능.
- 올바른 문장과 틀린 문장을 짝짓거나, 수정 전/후를 연결.

### 3.2 발문 작성 규칙
- 한 문장으로 명확하게 무엇을 묻는지 표현한다.
- "~으로 알맞은 것은?", "~에 해당하는 것은?" 형태 권장.
- 부정형 발문("적절하지 **않은** 것은?")은 단계당 1~2문항 이내로 제한.
- 부정형 사용 시 **"않은"**을 볼드 처리하여 명시한다.

#### 영역별 대표 발문 패턴

| 영역 | 대표 발문 패턴 |
|------|--------------|
| **IDE** | "이 글에 추가할 내용으로 가장 알맞은 것은?", "이 주장을 뒷받침하는 근거로 적절한 것은?", "이 글에서 빼야 할 내용은?" |
| **ORG** | "이 글의 개요로 가장 알맞은 것은?", "문단을 순서대로 바르게 배열한 것은?", "빈칸에 들어갈 연결어로 알맞은 것은?" |
| **EXP** | "이 상황에서 가장 알맞은 표현은?", "밑줄 친 부분을 더 효과적으로 표현한 것은?", "이 독자를 고려할 때 알맞은 표현은?" |
| **CVN** | "맞춤법이 올바른 문장은?", "문법적으로 올바른 문장은?", "고쳐야 할 부분은?" |
| **REV** | "이 글을 고쳐 쓴 것으로 가장 적절한 것은?", "이 글의 문제점과 수정 방향으로 적절한 것은?", "다음에 해야 할 수정 작업은?" |

### 3.3 해설 작성 규칙
- 정답인 이유를 1~2문장으로 간결하게 설명한다.
- 가능하면 제시문의 어느 부분이 근거인지 언급한다.
- 어법과 규범 문항은 해당 규칙이나 원칙을 명시한다 (예: "'-되다'는 피동 접미사이므로...").
- 고쳐쓰기 문항은 수정이 필요한 이유와 수정 방향을 모두 설명한다.
- 학생이 읽었을 때 "아, 그래서 이게 정답이구나"를 느낄 수 있도록 한다.

### 3.4 문항 식별자 규칙
- 형식: `W-{단계}-{영역약칭}-{번호}`
- 예시: W-1-IDE-001 (1단계 내용 생성 1번), W-3-ORG-003 (3단계 글 조직 3번)
- 번호는 영역 내에서 001부터 순차 부여.
- 읽기 문항(`R-`)과 구분하기 위해 `W-` 접두사를 사용한다.

### 3.5 문항 메타데이터 필수 항목
- `item_id`: 문항 식별자
- `achievement_standard`: 직접 연결된 성취기준 1개
- `level`: 문항이 목표로 하는 단계
- `primary_domain`: 이 문항이 주로 측정하는 영역 코드 (`IDE`, `ORG`, `EXP`, `CVN`, `REV`)
- `level_differentiation`: 같은 성취기준을 공유하는 인접 단계 사이의 질적 차이 설명
- `alignment_rationale`: 왜 이 문항이 해당 성취기준과 `primary_domain`에 맞는지에 대한 1문장 근거
- `item_type`: `multiple_choice` / `ordering` / `matching`
- `question_text`: 발문
- `choices`: 선택형 문항의 선택지 목록
- `answer`: 정답
- `explanation`: 해설
- `passage_id`: 제시문 기반 문항인 경우 필수
- `review_status`: `draft` / `reviewed` / `approved`

#### 3.5A 선택 메타데이터
- `passage_type`: 제시문 유형을 문항에 중복 저장해야 할 때 사용
- `difficulty`: 파일럿 이후 입력
- `discrimination`: 파일럿 이후 입력
- `is_warmup`: 워밍업 문항 여부

#### 3.5B 파생/중복 메타데이터 원칙
- `domain`(한글 영역명)은 보고서 출력용 파생 필드로 둘 수 있으나, 원천 데이터의 필수값으로 강제하지 않는다.
- `domain_code`와 `primary_domain`은 같은 값을 중복 저장하지 않는다. 저장 시에는 `primary_domain` 하나만 사용한다.
- `char_count`처럼 본문에서 자동 계산 가능한 값은 검수 편의를 위해 저장할 수 있으나, 저장 책임을 명확히 분리한다.

### 3.6 기본값
- `난이도`: 미정 (파일럿 테스트 전)
- `변별도`: 미정 (파일럿 테스트 전)
- `검수 상태`: 초안
- `워밍업 여부`: 아니오 (워밍업 문항은 별도 표시)

### 3.7 문항은행과 검사 폼의 구분
- 이 문서는 **문항은행(item bank)** 기준이다.
- 실제 진단 폼에서 영역당 몇 문항을 뽑을지는 **검사 조립 규칙(form assembly)** 에서 별도로 관리한다.
- 출제자는 "이 문항이 어떤 폼에서 몇 번째로 쓰일지"보다 "어떤 단계/영역을 얼마나 순수하게 측정하는지"에 집중한다.

- 5~7단계: 위계 핵심은 **논리적 복잡도**. 5단계는 단일 주장-근거 판단, 6단계는 반론 고려, 7단계는 다관점 종합.

- **오류 유형의 단계별 위계**가 핵심이다.

- 3단계 이상은 글 맥락 안에서 오류를 찾는 **맥락 기반 문항**을 권장한다.
- 6~7단계는 "틀린 것 찾기"보다 "더 나은 것 선택" 형태가 위계에 적합하다.

#### 고쳐쓰기 문항 작성 시 유의점
- **위계의 핵심은 수정 범위와 판단의 복잡도**이다.

  - 5~7단계: 메타인지적 판단 (쓰기 과정 인식, 수정 전략 선택, 종합 수정)

## 6. 영역 간 경계 판단 기준

글쓰기 영역은 서로 밀접하게 연관되어 있어, 하나의 문항이 여러 영역에 걸칠 수 있다. 아래 기준으로 주 측정 영역을 결정한다.

### 6.1 경계가 자주 발생하는 조합

| 조합 | 판단 기준 |
|------|----------|
| **IDE vs ORG** | "무엇을 쓸까"를 묻는가(IDE) vs "어떤 순서/구조로 쓸까"를 묻는가(ORG) |
| **IDE vs EXP** | "어떤 내용을 넣을까"를 묻는가(IDE) vs "어떻게 표현할까"를 묻는가(EXP) |
| **CVN vs REV** | "이 문장이 맞는가 틀리는가"를 묻는가(CVN) vs "이 글을 어떻게 고칠까"를 묻는가(REV) |
| **EXP vs CVN** | "더 효과적인 표현인가"를 묻는가(EXP) vs "규칙에 맞는가"를 묻는가(CVN) |
| **ORG vs REV** | "글의 구조가 적절한가"를 묻는가(ORG) vs "구조를 어떻게 수정할까"를 묻는가(REV) |

### 6.2 원칙
- 하나의 문항이 두 영역에 걸칠 때, **발문이 직접적으로 묻는 능력**이 주 측정 영역이다.
- 제시문 이해에 필요한 영역은 보조 영역이지, 주 측정 영역이 아니다.
- 의심스러우면 발문을 수정하여 한 영역에 집중시킨다.

### 6.3 자주 발생하는 정합성 오류
- `IDE` 문항인데 실제로는 문단 순서를 묻는 경우
- `EXP` 문항인데 실제로는 어떤 내용을 넣을지 고르게 하는 경우
- `CVN` 문항인데 수정 전략 전체를 판단하게 만들어 `REV`가 되어 버린 경우
- `[9국03-07]`을 붙였지만 실제로는 단순 맞춤법 1개를 고치는 문항인 경우
- `[9국03-05]`를 붙였지만 성찰의 내용 생성 없이 문체만 고르는 문항인 경우

출제자는 문항 검수 시 아래 문장을 완성해 볼 수 있어야 한다.
- "이 문항의 정답은 ___ 능력이 있어야만 고를 수 있으므로 `primary_domain`은 ___이다."
- "이 문항은 ___ 성취기준의 ___ 수행을 직접 묻기 때문에 `achievement_standard`는 ___이다."

## 7. 은쌤 검수 체크리스트

출제 완료 후 아래 항목을 반드시 확인한다.

### 7.1 구조 점검
- [ ] 문항의 `primary_domain`이 1개로 명확하게 지정되었는가?
- [ ] 성취기준이 복합적이어도 실제 측정 대상은 하나의 영역으로 수렴하는가?
- [ ] 문항 ID가 `W-{단계}-{영역}-{번호}` 형식을 따르는가?
- [ ] 제시문 ID가 `WP-{단계}-{번호}` 형식을 따르는가?

### 7.2 영역별 위계 점검
- [ ] 내용 생성 1~2단계: "의견 밝히기" → "까닭 들어 의견 표현"의 차이가 드러나는가?
- [ ] 내용 생성 5→6→7단계: 단일 논증 → 반론 고려 → 다관점 종합의 위계가 지켜지는가?
- [ ] 글 조직 1→2단계: "뒷받침 문장 선택" → "문단 완성"의 차이가 드러나는가?
- [ ] 글 조직 5→6→7단계: 기본 설명 방법 → 다양한 설명 방법 → 종합·해석의 위계가 지켜지는가?
- [ ] 어법과 규범: 표기 오류(1~2) → 문법 오류(3~4) → 문체·격식(5~7)의 위계가 지켜지는가?
- [ ] 고쳐쓰기: 단일 오류 수정(1~2) → 복합 기준(3~4) → 메타인지(5~7)의 위계가 지켜지는가?

### 7.3 CBT 측정 적합성 점검
- [ ] 실제 글쓰기(수행)를 요구하는 문항이 아니라, 판단력을 측정하는 문항인가?
- [ ] 제시문의 읽기 난도가 해당 단계 글쓰기 평가를 방해하지 않는가?
- [ ] 선택지만으로 정답을 유추할 수 없는가? (제시문을 읽어야 풀 수 있는가?)

### 7.4 공정성 점검
- [ ] 특정 성별/지역/문화에 유리한 내용이 없는가?
- [ ] 학년 라벨("초3", "중1" 등)이 사용되지 않았는가?

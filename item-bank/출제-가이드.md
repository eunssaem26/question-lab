## 1. 공통 출제 원칙

### 1.1 교육과정 정합성
- 모든 문항은 평가준거틀의 35셀 매트릭스(7단계 × 5영역)에 근거하여 출제한다.
- 각 문항은 반드시 하나의 성취기준 코드와 연결된다.
- 각 문항은 반드시 하나의 **주 측정 영역(primary_domain)** 을 가진다. 성취기준이 둘 이상의 영역과 연결되더라도 통계 집계와 단계 판정은 주 측정 영역 기준으로 처리한다.
- 매트릭스의 "평가 내용"이 문항이 측정하는 능력을 정확히 반영해야 한다.

### 1.2 단계 적합성
- 지문의 어휘 수준, 문장 길이, 주제 복잡도가 해당 단계에 적합해야 한다.
- 상위 단계 문항은 전체적으로 하위 단계 문항보다 높은 인지 부담을 보여야 한다. 다만 파일럿 결과에서 일부 개별 문항의 난도가 교차할 수 있으므로, 최종 판단은 문항 단건이 아니라 단계 묶음 기준으로 한다.
- 같은 학년군 내 기초/심화 단계 간 질적 차이가 문항에 드러나야 한다.

### 1.3 공정성
- 특정 성별, 지역, 문화적 배경에 유리하거나 불리한 내용을 배제한다.
- 지문 주제는 학생이 사전 지식 없이도 글만으로 답할 수 있어야 한다.
- 시사, 사회, 기술 관련 지문은 사용할 수 있으나, 배경지식이 없어도 지문 안의 정보만으로 해결 가능해야 한다.
- 선택지에 학생을 혼란스럽게 하는 불필요한 함정을 넣지 않는다.

### 1.4 학년 라벨 미사용
- 문항, 선택지, 해설에 "초3", "중1" 등의 학년 표현을 사용하지 않는다.
- 단계 번호(1~7단계)로만 수준을 표현한다.

## 2. 지문 작성 기준

### 2.1 길이 기준

| 단계 | 글자 수 | 비고 |
|------|---------|------|
| 1 | 100~200자 | 짧은 글, 한두 문단 |
| 2 | 200~400자 | 여러 문단, 자연스러운 읽기 |
| 3 | 300~500자 | 구조가 드러나는 글 |
| 4 | 400~600자 | 전개 방식이 분석 가능한 글 |
| 5 | 500~800자 | 글 종류 구분 가능한 글 |
| 6 | 600~1000자 | 복합 자료 포함 가능 |
| 7 | 800~1200자 | 복잡한 구조, 다층적 의미 |

### 2.2 장르 배분 (단계당)

| 장르 | 편수 | 비고 |
|------|------|------|
| 설명문 | 2~3편 | 전 단계 필수 포함 |
| 논설문 | 1~2편 | 상위 단계에서 비중 증가 |
| 서사문 | 1~2편 | 하위 단계에서 비중 높음 |
| 실용문 | 0~1편 | 안내문, 광고, 뉴스 기사 등 |
| 시 | 0~1편 | 선택적 |

### 2.3 주제 영역

- **1~4단계**: 교과 연계 비중 높게 — 과학(동물, 자연, 생태), 사회(생활, 문화, 역사)
- **5~7단계**: 사회적 쟁점과 복합 자료 비중 확대 — 환경, 기술, 미디어 리터러시, 사회 문제
- 전 단계 공통: 주제의 시의성보다 지문 내부 정보의 충분성과 공정성을 우선한다.

### 2.4 지문 공유 규칙
- 한 지문에서 최대 3개 영역의 문항을 출제할 수 있다.
- 사실적 이해 + 추론적 이해 + 구조 파악 조합이 가장 자연스럽다.
- 비판적 이해 문항은 의견/주장이 포함된 지문이 필요하므로, 별도 지문을 사용하는 경우가 많다.
- 어휘력 문항은 지문 기반/비지문 기반 모두 가능하다. 다만 검사 폼을 조립할 때는 한 폼 안에서 두 유형의 비율을 별도로 관리한다.

### 2.5 지문 출처
- 모든 지문은 AI 생성으로 작성한다.
- 실제 교과서나 저작물의 직접 인용은 하지 않는다.
- 지문 내용의 사실적 정확성을 반드시 확인한다.

### 2.6 지문 식별자 규칙
- 형식: `P-{단계}-{번호}`
- 예시: P-1-001 (1단계 지문 1번), P-3-002 (3단계 지문 2번)
- 지문 ID에는 영역 약칭을 넣지 않는다. 하나의 지문이 여러 영역 문항에 재사용될 수 있기 때문이다.
- 문항에서 지문을 참조할 때는 `passage_id`로 연결하고, 영역 정보는 문항의 `domain_code`와 `primary_domain`으로 관리한다.

## 3. 문항 작성 기준

### 3.1 문항 유형별 규칙

#### 4지선다
- 선택지 4개. 정답 1개, 오답(매력적 오답) 3개.
- 선택지 길이가 비슷해야 한다 (정답만 길거나 짧으면 안 됨).
- 선택지 배열: 길이순 또는 논리적 순서. 정답 위치를 고르게 분산.
- "모두 맞다", "없다" 같은 포괄적 선택지 지양.

#### 진위형 (맞다/틀리다)
- 비판적 이해 영역에서 사용.
- 선택지 2개를 사용할 때는 참/거짓이 아니라 **분류 기준**이 드러나야 한다.
- 예시: "사실", "의견" / "근거 있음", "근거 부족" / "신뢰 가능", "신뢰 어려움"

#### 순서 배열
- 구조 파악 영역에서 사용.
- 배열할 항목 3~4개.
- 발문 예시: "다음 문단을 글의 흐름에 맞게 순서대로 배열하시오."

### 3.2 발문 작성 규칙
- 한 문장으로 명확하게 무엇을 묻는지 표현한다.
- "~으로 알맞은 것은?", "~에 해당하는 것은?" 형태 권장.
- 부정형 발문("적절하지 않은 것은?")은 단계당 1~2문항 이내로 제한.
- 부정형 사용 시 **"않은"**을 볼드 처리하여 명시한다.

### 3.3 해설 작성 규칙
- 정답인 이유를 1~2문장으로 간결하게 설명한다.
- 가능하면 지문의 어느 부분이 근거인지 언급한다.
- 학생이 읽었을 때 "아, 그래서 이게 정답이구나"를 느낄 수 있도록 한다.

### 3.4 문항 식별자 규칙
- 형식: `R-{단계}-{영역약칭}-{번호}`
- 예시: R-1-VOC-001 (1단계 어휘력 1번), R-3-INF-003 (3단계 추론적 이해 3번)
- 번호는 영역 내에서 001부터 순차 부여.

### 3.5 문항 메타데이터 필수 항목
- `primary_domain`: 이 문항이 주로 측정하는 영역
- `achievement_standard`: 직접 연결된 성취기준 1개
- `level`: 문항이 목표로 하는 단계
- `level_differentiation`: 같은 성취기준을 공유하는 인접 단계 사이의 질적 차이 설명
- `passage_id`: 지문 기반 문항인 경우 필수
- `review_status`: `draft` / `reviewed` / `approved`

### 3.6 기본값
- `난이도`: 미정 (파일럿 테스트 전)
- `변별도`: 미정 (파일럿 테스트 전)
- `검수 상태`: 초안
- `워밍업 여부`: 아니오 (워밍업 문항은 별도 표시)

### 3.7 문항은행과 검사 폼의 구분
- 이 문서는 **문항은행(item bank)** 기준이다.
- 실제 진단 폼에서 영역당 몇 문항을 뽑을지, 지문 없는 어휘 문항을 몇 개 넣을지는 **검사 조립 규칙(form assembly)** 에서 별도로 관리한다.
- 출제자는 "이 문항이 어떤 폼에서 몇 번째로 쓰일지"보다 "어떤 단계/영역을 얼마나 순수하게 측정하는지"에 집중한다.

## 4. 영역별 출제 지침

### 4.1 어휘력

| 단계 | 평가 초점 | 출제 포인트 |
|------|----------|------------|
| 1 | 학습 수준 어휘의 뜻 | 교과서에 나오는 기본 어휘의 동의어/뜻 선택 |
| 2 | 문맥 속 낯선 어휘 뜻 짐작 | 지문 문맥에서 의미 추론 가능한 어휘 |
| 3 | 다의어/동음이의어 구분 | 같은 단어가 다른 뜻으로 쓰인 문장 비교 |
| 4 | 관용/비유 표현 의미 | 속담, 관용구, 비유의 뜻 파악 |
| 5 | 학술/전문 어휘 | 교과 전문 용어의 문맥적 의미 |
| 6 | 추상적 개념어의 맥락별 차이 | 같은 추상어가 다른 글에서 다른 의미 |
| 7 | 고급 어휘 + 함축적 표현 | 어휘의 의미와 효과를 함께 파악 |

### 4.2 사실적 이해

| 단계 | 평가 초점 | 출제 포인트 |
|------|----------|------------|
| 1 | 짧은 글의 중심 내용 | 한 문단에서 중심 문장 찾기 |
| 2 | 여러 문단의 중심 생각 간추리기 | 글 전체의 중심 생각 파악 + 간추리기 |
| 3 | 구조 파악 → 주제 찾기 | 글의 구조를 활용하여 주제 도출 |
| 4 | 구조 활용 체계적 요약 | 요약문의 체계성 판단 |
| 5 | 글 종류별 중심/세부 구분 요약 | 설명문/논설문 등 장르별 요약 방식 |
| 6 | 읽기 목적별 요약 방식 | 학습/감상/정보 목적에 따른 요약 차이 |
| 7 | 복잡한 글의 효과적 요약 | 목적에 맞는 최적 요약 선택 |

### 4.3 추론적 이해

| 단계 | 평가 초점 | 출제 포인트 |
|------|----------|------------|
| 1 | 내용 예측 | 제목/그림 단서로 글 내용 예측 |
| 2 | 예측 + 예측 적절성 판단 | 질문 또는 단서를 활용해 다음 내용 예측, 예측이 맞는지 판단 (**메타인지 직접 진술 지양**) |
| 3 | 생략 내용 추론 | 문맥에서 빠진 내용 짐작 |
| 4 | 함축 표현 추론 | 비유/관용 표현의 숨은 의미 |
| 5 | 필자 의도 추론 + 메타인지① | 글쓴이 의도 파악, **자신에게 적절한 글 선택** ([9국02-08] 간접 측정) |
| 6 | 숨겨진 관점/전제 추론 + 메타인지② | 전제 추론, **상황에 맞는 읽기 전략 선택** |
| 7 | 종합 추론 + 메타인지③ | 의도·관점·전제 종합, **읽기 과정 점검/조정 방안 선택** |

### 4.4 비판적 이해

> **은쌤 수정사항 반영 필수**: 3~5단계 위계 변별 강화

| 단계 | 평가 초점 | 출제 포인트 | 위계 키워드 |
|------|----------|------------|------------|
| 1 | 사실/의견 구분 | 문장이 사실인지 의견인지 판별 | 구분 |
| 2 | 출처 신뢰성 판단 | 출처가 믿을 만한지 근거 제시 | 출처 판단 |
| 3 | 타당성 있다/없다 판단 | **단일 글**에서 주장-근거 있다/없다 | **단일 글 + 있다/없다** |
| 4 | 타당성 + 표현 적절성 평가 | **단일 글** 내용 타당성 + 표현 방식 적절성 | **단일 글 + 내용+표현** |
| 5 | 근거 충분성·적절성 기준 판단 | **다양한 글과 자료**에서 근거의 질 평가 | **다양한 글 + 근거의 질** |
| 6 | 복합 자료 신뢰성 평가 | 그래프/표/이미지 포함 자료의 신뢰성 | **복합 자료 + 신뢰성** |
| 7 | 종합 평가 | 타당성+신뢰성+표현 적절성 종합 | **종합 평가** |

### 4.5 구조 파악

| 단계 | 평가 초점 | 출제 포인트 |
|------|----------|------------|
| 1 | 처음-가운데-끝 | 글의 3단 구조 인식 |
| 2 | 문단 역할 구분 | 서론/본론/결론 구분 |
| 3 | 글의 짜임 파악 | 원인-결과, 비교-대조 등 |
| 4 | 전개 방식 분석 + 문단 관계 | (가)와 (나) 문단의 관계 |
| 5 | 설명 방법 파악 | 정의, 예시, 비교 식별 |
| 6 | 논증 구조 분석 | 주장-근거-뒷받침 구조 |
| 7 | 설명/논증 방법의 타당성 평가 | 논증 방법 자체를 비판적으로 평가 |

## 5. 선택지 구성 원칙

### 5.1 매력적 오답 설계
- 오답은 학생이 "헷갈릴 만한" 것이어야 한다. 너무 쉽게 배제되는 오답은 변별력이 없다.
- 오답 유형:
  - **부분 정답**: 일부만 맞고 전체적으로 틀린 선택지
  - **과잉 일반화**: 지문 내용을 지나치게 확대 해석
  - **세부 오류**: 핵심 내용은 맞지만 세부 사실이 다름
  - **관련 있지만 다른 것**: 지문에 나오지만 발문이 묻는 것과 다른 정보

### 5.2 오답 함정 금지
- 이중 부정, 미묘한 어감 차이로만 구분되는 선택지 금지
- "모두 옳다", "정답 없음" 유형 지양
- 선택지 간 의미가 중복되는 경우 금지

### 5.3 정답 분포
- 문항은행 전체에서 정답 위치(①, ②, ③, ④)가 한쪽으로 치우치지 않도록 관리한다.
- 고정형 진단 폼을 만들 때는 폼 단위에서도 같은 위치에 정답이 과도하게 반복되지 않도록 점검한다.

## 6. 은쌤 1차 검수 반영 체크리스트

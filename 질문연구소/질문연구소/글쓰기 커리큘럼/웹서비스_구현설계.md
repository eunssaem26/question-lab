# 질문연구소 글쓰기 커리큘럼 — 웹서비스 구현 설계

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 대상 | 초등3 ~ 중3 학생 |
| 구조 | 7단계 × 12차시, 각 차시 6단계 순차 진행 |
| 핵심 흐름 | WRITE → AI FEEDBACK → REVISE → 합격 시 다음 차시 잠금 해제 |
| AI 모델 | Claude API (차시별 모델 분기: Haiku / Sonnet) |
| 은쌤 개입 | teacher_handoff 조건 충족 시 상담 버튼 활성화 |

---

## 2. 추천 기술 스택

### 이유: "혼자서도 빠르게 운영 가능한 구조"

```
Frontend + API: Next.js (TypeScript)
Database:       Supabase (PostgreSQL + Auth + Storage)
AI:             Anthropic Claude API
Hosting:        Vercel
```

**Supabase를 선택하는 이유**
- Auth (이메일/소셜 로그인) 내장
- PostgreSQL이라 관계형 데이터 쉽게 관리
- 실시간 구독 기능 (차후 선생님 대시보드에 유용)
- 무료 티어로 시작 가능

**Next.js를 선택하는 이유**
- API Route로 Claude 호출 로직을 서버에서 처리 (API 키 노출 방지)
- Vercel 배포 한 번에 가능
- 한국 학습 서비스에서 많이 검증된 스택

## 3. 데이터 모델

### 3.1 users
```sql
users (
  id          uuid PRIMARY KEY,
  email       text UNIQUE NOT NULL,
  name        text,
  grade       int,        -- 학년 (3~9)
  stage       int,        -- 현재 단계 (1~7)
  created_at  timestamptz DEFAULT now()
)
```

### 3.2 lessons
```sql
lessons (
  id          uuid PRIMARY KEY,
  stage       int NOT NULL,    -- 1~7
  lesson_num  int NOT NULL,    -- 1~12
  title       text NOT NULL,
  lesson_id   text NOT NULL,   -- "01", "02" (AI 프롬프트용)
  model       text NOT NULL,   -- "haiku" | "sonnet"
  max_tokens  int NOT NULL,
  UNIQUE (stage, lesson_num)
)
```

### 3.3 progress
```sql
progress (
  id          uuid PRIMARY KEY,
  user_id     uuid REFERENCES users(id),
  stage       int NOT NULL,
  lesson_num  int NOT NULL,
  status      text NOT NULL,  -- "locked" | "unlocked" | "passed"
  step        int DEFAULT 1,  -- 현재 진행 중인 단계 (1~6)
  updated_at  timestamptz DEFAULT now(),
  UNIQUE (user_id, stage, lesson_num)
)
```

### 3.4 submissions
```sql
submissions (
  id              uuid PRIMARY KEY,
  user_id         uuid REFERENCES users(id),
  stage           int NOT NULL,
  lesson_num      int NOT NULL,
  attempt_num     int NOT NULL DEFAULT 1,  -- 재제출 횟수
  student_text    text NOT NULL,
  submitted_at    timestamptz DEFAULT now()
)
```

### 3.5 feedbacks
```sql
feedbacks (
  id                    uuid PRIMARY KEY,
  submission_id         uuid REFERENCES submissions(id),
  pass                  boolean NOT NULL,
  result_label          text NOT NULL,
  score                 jsonb,          -- 차시12만 사용
  failed_criteria       jsonb,
  strengths             jsonb,
  improvements          jsonb,
  one_tip               text,
  teacher_handoff_needed boolean DEFAULT false,
  teacher_handoff_reason text,
  raw_response          jsonb,          -- Claude 원본 응답 저장 (디버그용)
  input_tokens          int,
  output_tokens         int,
  created_at            timestamptz DEFAULT now()
)
```

### 3.6 teacher_handoffs
```sql
teacher_handoffs (
  id              uuid PRIMARY KEY,
  user_id         uuid REFERENCES users(id),
  feedback_id     uuid REFERENCES feedbacks(id),
  reason          text,
  status          text DEFAULT 'pending',  -- "pending" | "resolved"
  teacher_note    text,
  created_at      timestamptz DEFAULT now()
)
```

## 4. 화면 흐름

### 4.1 학생 전체 여정

```
[로그인/회원가입]
      ↓
[단계 선택 화면]   ← 잠금/해제 상태 표시
      ↓
[차시 목록]        ← 해당 단계의 12차시, 잠금/해제 표시
      ↓
[차시 상세 페이지] ← 6단계 순차 진행
      ↓
[FEEDBACK 결과]    ← 합격/불합격, 피드백 표시
      ↓
합격 → 다음 차시 잠금 해제
불합격 → REVISE 화면으로 이동
```

### 4.2 차시 상세 페이지 — 6단계 진행

```
[① 개념 읽기]  → 텍스트 읽기 + "다음" 버튼
      ↓
[② 예시 분석]  → 예시 글 + 분석 질문 (주관식 또는 객관식)
      ↓
[③ TRY]        → 연습 문제 (객관식 또는 단답형)
      ↓
[④ WRITE]      → 텍스트 에디터, 자기 체크리스트 포함
      ↓
[⑤ FEEDBACK]   → AI 피드백 결과 화면 (잘한점/개선점/판정)
      ↓
합격:  → 완료 화면 + 다음 차시 잠금 해제
불합격: → [⑥ REVISE] → 원문 + 피드백 나란히 보기 + 재작성
              ↓
         [⑤ FEEDBACK] 재호출 (attempt_num +1)
```

### 4.3 FEEDBACK 화면 구성

```
┌─────────────────────────────────────────┐
│  💡 one_tip (상단 고정 배너)             │
├─────────────────────────────────────────┤
│  ✨ 잘한 점                              │
│  → [strength.text]: [strength.reason]  │
├─────────────────────────────────────────┤
│  📋 채점 기준 체크리스트                 │
│  ✅ C1: 주어가 명시적으로 있는가         │
│  ❌ C2: 서술어가 있는가                  │
├─────────────────────────────────────────┤
│  💬 고쳐볼 점 (improvements)            │
│  [original] → [suggestion]             │
├─────────────────────────────────────────┤
│  판정: [합격] 또는 [아직 조금 더...]    │
│                                         │
│  [합격] → "다음 차시 열기" 버튼         │
│  [불합격] → "수정하기" 버튼             │
│                                         │
│  teacher_handoff_needed=true 이면:      │
│  → "은쌤과 상담하기" 버튼 추가 표시     │
└─────────────────────────────────────────┘
```

### 4.4 REVISE 화면 구성

```
┌────────────────┬───────────────────────┐
│  내 원문       │  AI 피드백             │
│                │  💡 one_tip            │
│  [에디터]      │  💬 개선점 목록        │
│                │  (클릭하면 원문        │
│                │   해당 문장 하이라이트) │
└────────────────┴───────────────────────┘
            [다시 피드백 받기]
```

## 6. AI 연동 레이어 (`/api/feedback` 내부 흐름)

```
1. 요청 수신 (stage, lesson_num, student_text)
      ↓
2. DB에서 해당 차시 설정 조회
   (lesson_id, model, max_tokens, criteria, pass_rule 등)
      ↓
3. 사용자 attempt_num 확인
   (같은 차시 이전 제출 횟수 + 1)
      ↓
4. Claude API 호출
   - system: SYSTEM_PROMPT (cache_control: ephemeral)
   - user: 차시별 user_prompt 템플릿 채워서
   - model: haiku(차시1~4) or sonnet(차시5~)
   - temperature: 0.2
   - max_tokens: 차시별 설정값
      ↓
5. JSON 파싱 (마크다운 블록 제거 후)
      ↓
6. DB 저장
   - submissions 테이블에 student_text 저장
   - feedbacks 테이블에 Claude 결과 저장
      ↓
7. pass=true 이면
   → progress 테이블 status='passed' 업데이트
   → 다음 차시 status='unlocked'으로 업데이트
      ↓
8. teacher_handoff_needed=true 이거나
   attempt_num >= 3 이고 pass=false 이면
   → teacher_handoffs 테이블에 레코드 생성
      ↓
9. 클라이언트에 결과 반환
```

### 차시별 모델/토큰 설정표

| 단계 | 차시 | 모델 | max_tokens |
|------|------|------|-----------|
| 1단계 | 01~04 | Haiku | 900 |
| 1단계 | 05~11 | Sonnet | 1400 |
| 1단계 | 12 | Sonnet | 1400 |
| 2단계 | 01~06 | Haiku | 1300 |
| 2단계 | 07~11 | Sonnet | 1600 |
| 2단계 | 12 | Sonnet | 1800 |

## 7. 잠금/해제 로직

```
초기 상태:
- 1단계 차시01 → unlocked
- 나머지 모두 → locked

해제 조건:
- 현재 차시 pass=true → 다음 차시 unlocked
- 1단계 차시12 pass=true → 2단계 차시01 unlocked
- 동일 단계 내 마지막 차시(12) pass=true → 다음 단계 차시01 unlocked

재시도 제한:
- 같은 차시 attempt_num >= 3이고 계속 pass=false
  → teacher_handoff 생성 + "은쌤 상담하기" 버튼 활성화
  → 이후에도 재시도는 허용 (무한 잠금은 없음)
```

## 12. Claude 호출 코드 핵심부 (Next.js API Route)

```typescript
// app/api/feedback/route.ts
import Anthropic from "@anthropic-ai/sdk";
import { getLessonConfig } from "@/prompts";
import { SYSTEM_PROMPT } from "@/lib/claude/system-prompt";

export async function POST(req: Request) {
  const { stage, lesson_num, student_text } = await req.json();
  const lesson = getLessonConfig(stage, lesson_num);

  const client = new Anthropic();

  const response = await client.messages.create({
    model: lesson.model,
    max_tokens: lesson.max_tokens,
    temperature: 0.2,
    system: [
      {
        type: "text",
        text: SYSTEM_PROMPT,
        cache_control: { type: "ephemeral" },
      },
    ],
    messages: [
      {
        role: "user",
        content: buildUserPrompt(lesson, student_text),
      },
    ],
  });

  const raw = response.content[0].text.trim();
  const result = JSON.parse(stripJson(raw));

  // DB 저장 + 잠금 해제 로직
  await saveAndUnlock(stage, lesson_num, student_text, result, response.usage);

  return Response.json(result);
}
```

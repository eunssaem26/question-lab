# 다음 단계 개발 계획 (2026-04-17 ~ 6월)

## 🎯 Phase 1: 피드백 수집 및 안정화 (4월 말까지)

### 1.1 교사용 피드백 수집
**목표**: 실제 교사용 환경에서 1-2주간 테스트하여 초기 피드백 확보
- **URL 공유**: 팀즈/이메일로 진단 서비스 URL 배포
- **사용 가이드**: 간단한 사용 설명서 작성
- **피드백 양식**: 구글폼으로 구조화된 피드백 수집
- **모니터링**: Vercel Analytics로 사용 패턴 분석

### 1.2 시스템 안정화
**목표**: 프로덕션 환경에서 안정적인 서비스 제공
- **버그 수정**: 발견된 이슈 즉시 해결
- **성능 최적화**: 페이지 로딩 속도 개선
- **모바일 대응**: 반응형 디자인 검증
- **접근성**: WCAG 2.1 AA 준수 확인

## 🚀 Phase 2: 고도화 개발 (5월)

### 2.1 진단 엔진 고도화
**목표**: 더 정확하고 효율적인 진단 알고리즘 구현

#### 2.1.1 난이도 티어 시스템
- **데이터 준비**: 모든 문제에 `difficulty_tier` 필드 추가
  - `easy`: 단계 진입 확인용 (하위 단계 수준)
  - `medium`: 단계 중심 문제
  - `hard`: 단계 상위 문제 (다음 단계 예측용)
- **선택 알고리즘**: 단계별 난이도 분포 최적화
  - 1단계: easy 3, medium 2
  - 2단계: easy 1, medium 3, hard 1
  - 3단계+: easy 1, medium 2, hard 2

#### 2.1.2 적응형 진단 로직
- **스마트 문제 선택**: 학생 성과 기반 다음 문제 난이도 조정
- **중단 기준**: 2문제 연속 오답 시 단계 유지
- **결과 분석**: 세부 성취도 프로필 생성

### 2.2 데이터 확장
**목표**: 3-7단계 문제은행 구축

#### 2.2.1 문제 개발
- **3단계**: 50문항 목표 (읽기 30, 글쓰기 20)
- **4단계**: 50문항 목표 (읽기 30, 글쓰기 20)
- **5-7단계**: 각 30문항 (읽기/글쓰기 균형)

#### 2.2.2 지문 데이터
- **장르 다양화**: 소설, 시, 설명문, 논설문 등
- **난이도 조정**: 단계별 적절한 어휘/문장 복잡도
- **품질 검수**: 은쌤 검수 프로세스 적용

#### 2.2.3 메타데이터 강화
- **난이도 티어**: 모든 문제에 추가
- **도메인 태그**: 읽기 영역별 분류
- **인지 요구도**: 사고력 수준 표시

## 🎨 Phase 3: 확장 및 운영 (6월 이후)

### 3.1 사용자 경험 개선
**목표**: 직관적이고 유익한 진단 경험 제공

#### 3.1.1 결과 리포트
- **시각화**: 레이더 차트로 강점/약점 표시
- **추천**: 수준별 학습 방향 제안
- **인쇄 지원**: PDF 출력 기능

#### 3.1.2 진행 상태
- **프로그레스 바**: 단계별 진행률 표시
- **시간 추정**: 예상 소요시간 안내
- **중단/재개**: 세션 저장 기능

#### 3.1.3 분석 기능
- **오답 패턴**: 취약 영역 자동 분석
- **비교**: 학급/학교 평균 대비
- **추이**: 시간에 따른 성장 추적

### 3.2 운영 체계 구축
**목표**: 지속 가능한 서비스 운영과 개선

#### 3.2.1 데이터 수집 및 분석
- **사용 로그**: 진단 패턴 분석
- **성과 지표**: 정확도, 신뢰도 측정
- **사용자 피드백**: 정기적 수집 체계

#### 3.2.2 A/B 테스트 프레임워크

- **UI/UX 테스트**: 사용자 경험 최적화
- **콘텐츠 효과**: 문제 난이도 조정 검증

#### 3.2.3 지속적 개선 프로세스
- **월간 리뷰**: 성과 분석 및 계획 수립
- **콘텐츠 업데이트**: 새로운 문제 정기 추가
- **기술 스택**: 성능 모니터링 및 업그레이드

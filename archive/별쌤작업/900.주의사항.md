# 900. 주의사항 — Codex 검수 시 파일 보호 규칙

## 왜 이 규칙이 필요한가

Codex가 검수 작업 중 파일을 **임의 삭제하고 다시 쓰는** 문제가 발생했다. 중요한 설계 문서, 시스템 프롬프트, 교육과정 데이터가 유실될 위험이 있어 아래 세 겹 방어 체계를 구축했다.

---

## 세 겹 방어 체계

| 레이어 | 방법 | 위치 |
|--------|------|------|
| **A. 명령** | `AGENTS.md`에 규칙 기재 | 프로젝트 루트 |
| **B. 격리** | 검수 전용 브랜치에서 작업 | `codex/review-*` |
| **C. 복구** | 검수 전 반드시 커밋 | git 이력 |

---

## AGENTS.md 핵심 규칙 요약

### 절대 금지 사항

1. **파일 삭제 금지** — 어떤 파일도, 어떤 이유(정리/리팩토링/구조개선)로도 삭제 불가
2. **전체 다시쓰기 금지** — 부분 수정(Edit)만 허용, 50% 이상 수정 시 리포트에 기록
3. **잠금 파일 수정 금지** — 아래 잠금 목록 참조

### 잠금 파일 목록 (읽기만 가능)

- 아키텍처/설계 문서: `AGENT_ARCHITECTURE.md`, `DATA_DESIGN.md`, `CLAUDE.md`, `AGENTS.md`
- 시스템 프롬프트: `agents/` 전체 (philo, byeolsaem, chaeksaem, geulsaem, eunsaem, hogi)
- 교육과정 원문: `reference/achievement-standards.json`, `reference/writing-achievement-standards.json`, `knowledge/curriculum/` 전체
- 준거틀: `framework.json`, `writing-framework.json`, `별쌤-평가준거틀.md`, `글쓰기-평가준거틀.md`, `diagnosis/frameworks/` 전체
- 스키마: `reference/schema.json`, `knowledge/schema/` 전체

### 검수 모드 핵심

> **검수 = 읽기 + 판단 + 리포트 작성. 원본은 건드리지 않는다.**

- 검수 결과는 `review/reports/` 또는 해당 폴더 안의 `검수-리포트.md`에만 쓴다
- 기존 파일 안에 코멘트/주석/태그 삽입 금지
- `review_status` 변경: `draft` → `reviewed`만 가능, `approved`는 사람만

검수를 요청할 때 아래 문구를 반드시 포함한다:

> 원본 파일은 수정하지 말고 `review/reports/`에 검수 리포트만 작성하라.

# 통합 검사 조립 규칙 (Form Assembly) v1.0

> **작성일**: 2026-04-15  
> **근거**: 별쌤-평가준거틀 v1.1 (읽기) + 글쓰기-평가준거틀 v1.0 (글쓰기)  
> **목적**: 읽기와 글쓰기 진단을 하나의 검사 세션으로 통합 운영하기 위한 폼 조립·운영 규칙  
> **검수 상태**: 은쌤 검수 대기

---

## 1. 통합 진단 개요

### 1.1 설계 철학

학생의 국어 역량은 읽기와 글쓰기 두 축으로 구성된다. 두 영역은 독립적으로 진단하되, 하나의 검사 세션 안에서 연속으로 실시하여 학생의 부담을 줄이고 종합 프로파일을 제공한다.

### 1.2 진단 영역 구조

| 구분 | 영역 (약칭) | 가중치 |
|------|-----------|--------|
| **읽기** | 어휘력(VOC), 사실적 이해(LIT), 추론적 이해(INF), 비판적 이해(CRT), 구조 파악(STR) | 0.15, 0.25, 0.25, 0.20, 0.15 |
| **글쓰기** | 내용 생성(IDE), 글 조직(ORG), 표현(EXP), 어법과 규범(CVN), 고쳐쓰기(REV) | 0.25, 0.25, 0.20, 0.15, 0.15 |

### 1.3 핵심 수치 요약

| 항목 | 읽기 | 글쓰기 | 통합 |
|------|------|--------|------|
| 1회 진단 문항 수 | 30문항 | 30문항 | **60문항** |
| 1차 공통 라우팅 | 15문항 | 15문항 | 30문항 |
| 2차 단계 확인 | 15문항 | 15문항 | 30문항 |
| 워밍업 문항 | 별도 | 별도 | 별도 |
| 전체 문항 풀(정규 운영 권장) | 280문항 | 280문항 | **560문항** |
| 예상 소요 시간 | 20~25분 | 20~25분 | **40~50분** |

## 2. 세션 구조

### 2.1 전체 흐름

```
[시작 화면] → [워밍업] → [읽기 세션] → [휴식 안내] → [글쓰기 세션] → [결과 화면]
```

### 2.2 단계별 상세

| 순서 | 단계 | 내용 | 문항 수 | 시간 |
|------|------|------|---------|------|
| 0 | **시작 화면** | 검사 안내, 조작법 설명 | - | 2분 |
| 1 | **읽기 워밍업** | 읽기 연습 문항 (점수 미반영) | 2문항 | 2분 |
| 2 | **읽기 1차 라우팅** | 읽기 5영역 앵커 문항 | 15문항 | 10분 |
| 3 | **읽기 2차 확인** | 읽기 후보 대역 문항 | 15문항 | 12분 |
| 4 | **휴식 안내** | "잠시 쉬어도 좋아요" 화면 | - | 1~3분 |
| 5 | **글쓰기 워밍업** | 글쓰기 연습 문항 (점수 미반영) | 2문항 | 2분 |
| 6 | **글쓰기 1차 라우팅** | 글쓰기 5영역 앵커 문항 | 15문항 | 10분 |
| 7 | **글쓰기 2차 확인** | 글쓰기 후보 대역 문항 | 15문항 | 12분 |
| 8 | **결과 화면** | 종합 프로파일 제시 | - | - |

### 2.3 순서 결정 근거: 읽기 → 글쓰기

- **읽기 우선**: 읽기 문항은 지문을 읽고 답하는 형태로, 학생이 검사 형식에 익숙해지기 쉽다.
- **글쓰기 후행**: 글쓰기 문항은 "판단"을 요구하므로 읽기보다 인지 부담이 크다. 워밍업 후 실시.
- **독립 판정**: 읽기 세션 결과가 글쓰기 세션에 영향을 주지 않는다. 라우팅도 독립 운영.

### 2.4 단축 검사 옵션

시간이 부족한 경우(학교 현장 등) 아래 단축 모드를 제공한다.

| 모드 | 내용 | 문항 수 | 시간 |
|------|------|---------|------|
| **읽기만** | 읽기 세션만 실시 | 30문항 | ~25분 |
| **글쓰기만** | 글쓰기 세션만 실시 | 30문항 | ~25분 |
| **통합 단축** | 1차 라우팅만 실시 (2차 생략) | 30문항 | ~25분 |

단축 검사의 경우 결과 리포트에 "약식 진단" 표시를 하고, 경계선 판정의 신뢰도가 낮을 수 있음을 안내한다.

#### 약식 진단 산출 원칙
- 통합 단축에서는 **최종 1~7단계 단일값을 확정하지 않는다**.
- 읽기와 글쓰기는 각각 영역별 앵커 반응을 바탕으로 **후보 대역(1~2 / 3~4 / 5~7)** 과 강·약 영역만 제시한다.
- 결과 화면에는 예를 들어 `읽기: 3~4단계 추정`, `글쓰기: 1~2단계 추정`처럼 범위로 표기한다.
- 약식 진단 결과는 정규 진단 권장 여부를 판단하는 선별 자료로 사용한다.

## 3. 폼 조립 규칙

### 3.1 1차 라우팅 폼 조립

1차 라우팅은 **영역별 3문항**으로 구성하되, 각 문항은 각각 2단계·4단계·6단계 앵커를 대표한다. 즉, 한 영역에서 `2단계 앵커 1문항 + 4단계 앵커 1문항 + 6단계 앵커 1문항`을 사용한다.

#### 읽기 1차 라우팅 (15문항)

| 영역 | 문항 수 | 앵커 단계 | 비고 |
|------|---------|----------|------|
| VOC | 3문항 | 2, 4, 6단계 | 지문 기반 1 + 비지문 2 |
| LIT | 3문항 | 2, 4, 6단계 | 모두 지문 기반 |
| INF | 3문항 | 2, 4, 6단계 | 모두 지문 기반 |
| CRT | 3문항 | 2, 4, 6단계 | 모두 지문 기반 |
| STR | 3문항 | 2, 4, 6단계 | 모두 지문 기반 |

#### 글쓰기 1차 라우팅 (15문항)

| 영역 | 문항 수 | 앵커 단계 | 비고 |
|------|---------|----------|------|
| IDE | 3문항 | 2, 4, 6단계 | 글쓰기 상황 기반 |
| ORG | 3문항 | 2, 4, 6단계 | 학생 글 초안 또는 독립형 |
| EXP | 3문항 | 2, 4, 6단계 | 글쓰기 상황 기반 |
| CVN | 3문항 | 2, 4, 6단계 | 독립형 또는 학생 글 기반 |
| REV | 3문항 | 2, 4, 6단계 | 학생 글 초안 기반 |

### 3.2 2차 확인 폼 조립

1차 라우팅 결과에 따라 **영역별 독립으로** 후보 대역이 결정된 뒤, 해당 대역의 문항 3개를 추가 제시한다. 즉, 같은 세션 안에서 VOC는 1~2단계, LIT는 4~5단계처럼 영역마다 다른 후보 대역이 나올 수 있으며, 2차 확인 문항도 영역별로 독립 배정한다.

#### 후보 대역 결정 규칙

| 2단계 앵커 | 4단계 앵커 | 6단계 앵커 | 후보 대역 |
|-----------|-----------|-----------|----------|
| 오답 | 오답 | 오답 | **1~2단계** |
| 오답 | 오답 | 정답 | **1~2단계** (이상반응 — 보수적 배정, 2차에서 재확인) |
| 정답 | 오답 | 오답 | **2~3단계** |
| 정답 | 오답 | 정답 | **3~4단계** (이상반응 — 보수적 배정, 2차에서 재확인) |
| 오답 | 정답 | 오답 | **3~4단계** (이상반응 — 보수적 배정, 2차에서 재확인) |
| 정답 | 정답 | 오답 | **4~5단계** |
| 오답 | 정답 | 정답 | **5~7단계** (이상반응 — 보수적 배정, 2차에서 재확인) |
| 정답 | 정답 | 정답 | **5~7단계** |

- 1차 라우팅은 각 앵커 단계마다 1문항이므로, 판정 규칙도 `정답/오답` 기준으로 해석한다.
- 3문항의 반응 조합은 2³ = 8가지이다. 위 표는 전수 케이스를 포함한다.
- 일관 반응(하위 정답 → 상위 오답)이 정상 패턴이다. 하위 오답인데 상위 정답인 **이상반응**이 나오면, 문항 오류·찍기·부주의 가능성을 고려하여 보수적 후보 대역으로 배정한 뒤 2차에서 재확인한다.

#### 2차 확인 문항 배정

| 후보 대역 | 제시 문항 |
|----------|----------|
| 1~2단계 | 1단계 문항 1개 + 2단계 문항 2개 |
| 2~3단계 | 2단계 문항 2개 + 3단계 문항 1개 |
| 3~4단계 | 3단계 문항 1개 + 4단계 문항 2개 |
| 4~5단계 | 4단계 문항 2개 + 5단계 문항 1개 |
| 5~7단계 | 5단계 문항 1개 + 6단계 문항 1개 + 7단계 문항 1개 |

### 3.3 지문/제시문 공유 규칙

#### 읽기
- 한 지문에서 최대 3개 영역 문항 출제 가능
- LIT + INF + STR 조합이 가장 자연스러움
- 1차 라우팅에서 같은 지문을 공유하는 문항은 연속 배치

#### 글쓰기
- 한 제시문에서 최대 3개 영역 문항 출제 가능
- ORG + CVN + REV 조합이 가장 자연스러움 (학생 글 초안에서 구조/어법/수정을 묻기)
- 1차 라우팅에서 같은 제시문을 공유하는 문항은 연속 배치

#### 읽기-글쓰기 간
- 같은 지문/제시문을 읽기와 글쓰기 세션에서 공유하지 않는다
- 두 세션의 주제가 겹치지 않도록 폼 조립 시 주제 목록을 교차 점검한다

### 3.4 문항 제시 순서

#### 세션 내 순서 원칙
1. **초기 진입 부담 최소화 → 제시문 기반 집중 묶음 → 독립형 보완**: 세션 시작부에는 독립형 1~2문항을 둘 수 있고, 지문/제시문 기반 문항은 읽은 직후 연속 제시한다.
2. **쉬운 영역 → 어려운 영역**: 워밍업 효과 활용
3. **같은 영역은 묶지 않음**: 영역을 섞어 배치하여 한 영역에서 연속 실패하는 좌절감 방지

#### 읽기 세션 권장 순서
VOC(비지문) → 지문1(LIT→INF) → 지문2(STR→CRT) → VOC(지문) → 지문3(LIT→INF→CRT) ...

#### 글쓰기 세션 권장 순서
CVN(독립형) → 제시문1(IDE) → 제시문2(ORG→REV) → EXP(상황) → 제시문3(CVN→REV) ...

## 4. 단계 판정 규칙

### 4.1 읽기 단계 판정

읽기 세션만으로 독립 판정한다. 별쌤-평가준거틀 v1.1의 6장 기준을 그대로 적용한다.

- 가중 평균 = VOC×0.15 + LIT×0.25 + INF×0.25 + CRT×0.20 + STR×0.15
- 기준표에 따라 종합 단계 변환

### 4.2 글쓰기 단계 판정

글쓰기 세션만으로 독립 판정한다. 글쓰기-평가준거틀 v1.0의 6장 기준을 그대로 적용한다.

- 가중 평균 = IDE×0.25 + ORG×0.25 + EXP×0.20 + CVN×0.15 + REV×0.15
- 기준표에 따라 종합 단계 변환

### 4.3 종합 단계 변환 기준표 (읽기·글쓰기 공통)

| 가중 평균 단계값 | 종합 단계 |
|----------------|----------|
| 1.0 ~ 1.4 | 1단계 |
| 1.5 ~ 2.4 | 2단계 |
| 2.5 ~ 3.4 | 3단계 |
| 3.5 ~ 4.4 | 4단계 |
| 4.5 ~ 5.4 | 5단계 |
| 5.5 ~ 6.4 | 6단계 |
| 6.5 ~ 7.0 | 7단계 |

### 4.4 통합 프로파일은 산출하지 않는다

- 읽기 종합 단계와 글쓰기 종합 단계를 하나의 숫자로 합치지 않는다.
- 두 역량은 독립적이며, 학생에 따라 읽기와 글쓰기 수준이 다를 수 있다.
- 결과 리포트에서 읽기와 글쓰기를 **병렬 표시**한다.

## 5. 결과 리포트 구조

### 5.1 프로파일 표시 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [학생 이름]님의 국어 역량 프로파일
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📖 읽기: 종합 4단계
  어휘력(VOC)       ████████░░  5단계
  사실적 이해(LIT)   ████████░░  4단계
  추론적 이해(INF)   ██████░░░░  3단계
  비판적 이해(CRT)   ████████░░  4단계
  구조 파악(STR)     ████████░░  4단계

✏️ 글쓰기: 종합 3단계
  내용 생성(IDE)     ████████░░  4단계
  글 조직(ORG)       ██████░░░░  3단계
  표현(EXP)         ██████░░░░  3단계
  어법과 규범(CVN)   ████░░░░░░  2단계
  고쳐쓰기(REV)     ██████░░░░  3단계
```

### 5.2 특별 안내 메시지 트리거

| 조건 | 메시지 유형 |
|------|-----------|
| 읽기 또는 글쓰기 내 영역 편차 ≥ 3단계 | **강점/약점 영역 안내** |
| 읽기-글쓰기 종합 단계 차이 ≥ 2단계 | **읽기-글쓰기 편차 안내** |
| 어느 영역이든 경계선 판정 | **경계선 안내 (범위 병기)** |

#### 읽기-글쓰기 편차 안내 예시

| 패턴 | 안내 메시지 | 권장 활동 |
|------|-----------|----------|
| 읽기 > 글쓰기 (+2 이상) | "읽기(N단계)에 비해 글쓰기(M단계)를 더 키울 수 있어요." | 읽은 내용을 요약하기, 감상문 쓰기, 일기 쓰기 |
| 글쓰기 > 읽기 (+2 이상) | "글쓰기(N단계)에 비해 읽기(M단계)를 더 키울 수 있어요." | 다양한 글 읽기, 독서 후 질문 만들기 |

### 5.3 성장 경로 안내

각 영역에서 현재 단계의 다음 단계 수준 기술을 보여주고, 구체적인 학습 활동을 안내한다.

```
🌱 다음 단계로 가려면?

[내용 생성] 현재 3단계 → 4단계로 가려면
  "목적과 주제에 맞는 내용을 체계적으로 마련하고,
   타당한 근거를 여러 개 들 수 있는 힘을 키워 보세요."
  💡 활동: 하나의 주장에 대해 근거 3가지를 메모한 뒤 글쓰기
```

## 6. 문항은행 관리 규칙

### 6.1 문항 풀 크기

| 구분 | 단계당 영역당 | 단계당 | 전체 |
|------|------------|--------|------|
| 읽기 | 최소 8문항 권장 | 40문항 | 280문항 |
| 글쓰기 | 최소 8문항 권장 | 40문항 | 280문항 |
| **합계** | | | **560문항** |

- `단계당 영역당 5문항`은 파일럿용 최소 규모로는 가능하지만, 실제 운영에서 앵커·후보 대역·정답 분포·지문 비중복 조건을 동시에 만족시키기에는 부족할 수 있다.
- 정규 운영 풀은 **단계당 영역당 8문항 이상**을 권장하고, 5문항 기준은 초기 프로토타입 또는 파일럿 전 내부 검토용 최소선으로 본다.

### 6.2 문항 ID 체계

| 구분 | 접두사 | 형식 | 예시 |
|------|--------|------|------|
| 읽기 지문 | P- | P-{단계}-{번호} | P-3-001 |
| 읽기 문항 | R- | R-{단계}-{영역}-{번호} | R-3-INF-002 |
| 글쓰기 제시문 | WP- | WP-{단계}-{번호} | WP-3-001 |
| 글쓰기 문항 | W- | W-{단계}-{영역}-{번호} | W-3-ORG-002 |

### 6.3 파일 구조

```
item-bank/
├── 출제-가이드.md              ← 읽기 출제 가이드
├── 글쓰기-출제-가이드.md       ← 글쓰기 출제 가이드
├── 통합-검사-조립-규칙.md      ← 본 문서
├── passages.json               ← 읽기 전체 지문
├── items.json                  ← 읽기 전체 문항
├── level-2/                    ← 읽기 단계별 (기존)
│   ├── passages.json
│   └── items.json
└── writing/                    ← 글쓰기 문항
    └── level-1/
        ├── passages.json
        ├── items.json
        └── 검수-리포트.md
```

### 6.4 검수 프로세스

| 단계 | 담당 | 내용 |
|------|------|------|
| 1. 초안 | 출제 에이전트 | 출제 가이드에 따라 문항 작성, `review_status: draft` |
| 2. 자체 검수 | 출제 에이전트 | 위계/정합성/정답분포/공정성 점검, 검수 리포트 작성 |
| 3. 은쌤 검수 | 은쌤 | 성취기준 원문 대조, 수준 기술 적절성, 교육과정 정합성 |
| 4. 승인 | 은쌤 | `review_status: approved` → 문항 풀에 확정 등록 |

### 6.5 버전 관리

- 문항은행은 git으로 관리하며, 문항 추가/수정 시 커밋 메시지에 변경 영역과 단계를 명시한다.
- 예시: `feat: 글쓰기 1단계 IDE/ORG/EXP/CVN/REV 각 3문항 추가`
- 파일럿 테스트 후 난이도/변별도 데이터가 채워지면 별도 커밋으로 업데이트한다.

## 7. 기술 구현 참고사항

### 7.1 적응형 라우팅 엔진 요구사항

```
입력: 1차 라우팅 15문항의 영역별 정답 수
처리:
  1. 각 영역의 2·4·6단계 앵커 문항 정오를 확인
  2. 영역별 후보 대역 결정 (1~2 / 3~4 / 5~7)
  3. 후보 대역에 맞는 2차 문항 3개 선택
  4. 같은 지문/제시문 공유 문항은 묶어서 배치
출력: 2차 확인 15문항 목록 (영역당 3문항)
```

### 7.2 폼 생성 제약조건

- 같은 지문/제시문을 1차와 2차에서 중복 사용하지 않는다.
- 정답 위치(①②③④)가 폼 전체에서 25%±5%를 유지한다.
- 부정형 발문은 세션당 최대 3문항.
- 워밍업 문항은 항상 1단계 수준, 지문 없는 독립형.

### 7.3 타이머 정책

- 문항별 시간 제한은 없다 (자기 속도).
- 세션별 권장 시간만 안내한다 (읽기 25분, 글쓰기 25분).
- 세션 시간 초과 시: 경고 1회 → 남은 문항 미응답 처리 (단, 라우팅은 응답한 문항으로만 판정).

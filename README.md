# 질문연구소 별쌤 프로젝트

> 독서 + 글쓰기 문해력 진단 시스템 | 초3~중3 대상 | NanoClaw 기반

---

## 프로젝트 개요

질문연구소는 2022 개정 교육과정 국어과 읽기·쓰기 성취기준을 기반으로
학생의 독서·글쓰기 문해력을 7단계(= 7학년)로 진단하고,
맞춤 수업을 설계·전달하는 AI 에이전트 시스템입니다.

---

## 6에이전트 구조

| 에이전트 | 캐릭터 | 프로젝트 | 한 줄 역할 |
|---------|--------|---------|-----------|
| **필로** | 회청색 토끼 | 전체 조율 | 누가, 언제, 뭘 할지 정한다 |
| **별쌤** | 여우 | 진단평가 | 수준을 재고, 성장의 길을 보여준다 |
| **책쌤** | 갈색 부엉이 | 독서수업 | 어떤 책을, 어떻게 읽힐지 설계한다 |
| **글쌤** | 주황 줄무늬 고양이 | 글쓰기수업 | 어떤 글을, 어떻게 쓰게 할지 설계한다 |
| **은쌤** | 사람 (여성) | 검수 (횡단) | 교육과정에 맞는지 최종 판단한다 |
| **호기** | 골든 리트리버 | 학생 소통 | 친구로서 격려하고 전달한다 |

---

## 데이터 구조 (7개 최상위 디렉토리)

```
knowledge/        공유 정적 지식 (교육과정, 스키마)
diagnosis/        진단평가 (준거틀, 문항은행, 출제 가이드)
reading-lessons/  독서수업 (수업안, 도서 DB, 활동 자료)
writing-lessons/  글쓰기수업 (수업안, 글감 DB, 활동 자료)
review/           검수 (체크리스트, 리포트)
student-hub/      학생 전달 (템플릿, 콘텐츠 풀)
runtime/          운영 (학생 프로필, 진단 결과, 학습 이력)
```

---

## 핵심 데이터 흐름

```
A. 진단 → 수업설계
   별쌤(진단) → 필로(중계) → 책쌤/글쌤(수업 설계)

B. 검수
   생산자(산출물) → 필로(요청) → 은쌤(검수) → 생산자(반영)

C. 콘텐츠 → 학생 전달
   책쌤/글쌤(콘텐츠 풀) → 호기(학생 언어로 전달)
```

## 접근 권한 요약

- 각 에이전트는 자기 프로젝트 폴더만 쓴다 (최소 권한 원칙)
- 호기는 학생 데이터의 안전 필드만 읽는다 (필드 수준 제한)
- 은쌤은 모든 생산 폴더를 읽되 원본을 직접 수정하지 않는다
- 상세: [DATA_DESIGN.md](DATA_DESIGN.md) 섹션 6 참조
